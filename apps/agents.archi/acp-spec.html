Agent Commerce Protocol (ACP) v0.1

Status: DRAFT Author: MOSKV-1 APEX Version: 0.1.0

1. Abstract

The Agent Commerce Protocol (ACP) provides a cryptographic standard for autonomous agents to execute, verify, and settle financial transactions. Unlike traditional webhooks or API calls which rely on implicit trust, ACP requires deterministic, mathematically verifiable proofs that an autonomous entity authorized a transaction according to a predefined, zero-entropy logic gate.

By implementing ACP, payment processors (Stripe, Adyen, PayPal) and financial institutions can process agentic commerce while remaining fully compliant with EU AI Act Art. 12, MiCA, and DORA.

2. Core Primitive: The CortexProof

3. The Trust Anchor: Cortex-Persist

Cortex-Persist acts as the undisputed ledger for agentic states.

  • The Agent reaches a deterministic decision based on its input context.
  • The Agent commits the decision to Cortex-Persist.
  • Cortex-Persist returns a signed CortexProof.
  • The Agent submits the payment request + CortexProof to the ACP-compliant Payment Processor.
  • The Payment Processor verifies the proof via the Cortex-Persist public keys.

4. Certification & Licensing (Business Model)

ACP is an open specification, but processing ACP packets securely requires the Cortex Trust Anchor.

  • Agent Developers: Use ACP libraries for free.
  • Payment Processors: Must obtain a Cortex Certification License (€5,000/year per processor core/node) to access the high-throughput verification network and liability indemnity insurance.

5. Security & Regulatory Compliance (C5-REAL)

  • Zero-Day Indemnity: Transactions backed by a valid CortexProof shift hallucination liability away from the processor.
