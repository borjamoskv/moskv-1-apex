# CORTEX-PERSIST — Data Model & Ontology (v10.0)

> **Status:** LIVE ORGANISM
> **Scope:** Graph Schema & Stream Payloads

## 1. Graph Ontology (Neo4j)

The autopoiesis engine generates structural invariants dynamically. The base ontological primitives are:

### Nodes
- **`(:MemoryNode)`**: A fundamental unit of retained context.
  - `id` (String): Unique UUID or SHA256 of the event that spawned it.
  - `entropy` (Float): Signal density. Values closer to 1.0 indicate high volatility; values closer to 0.0 indicate crystallized invariants.
  - `content` (String): The structural payload (often YAML/JSON).
  - `lastUpdated` (Integer): Unix timestamp.

- **`(:BrainRegion)`**: Represents an active Swarm Worker in the graph.
  - `name` (String): Region identifier (e.g., `Vision`, `Reasoning`, `Autopoiesis`).
  - `status` (String): `ONLINE`, `SLEEP`, `DEAD`.

### Edges (Relationships)
- **`[:DERIVED_FROM {weight: Float}]`**: Tracks causality between memory nodes.
- **`[:SYNTHESIZED_BY]`**: Links a `MemoryNode` to the `BrainRegion` that emitted it.
- **`[:CONNECTED_TO {latency: Integer}]`**: Maps network topology between `BrainRegion` instances.

## 2. Event Payload Schema (NATS JetStream)

Every message circulating in the swarm strictly adheres to the following wrapper schema:

```typescript
interface CortexEvent {
    // Generated by EventBus
    hash: string;         // SHA-256 of (prevHash + payload)
    prevHash: string;     // Hash of the immediate predecessor event
    timestamp: number;    // Date.now()

    // Application Payload
    payload: {
        sourceRegion: string;  // e.g. "Reasoning"
        type: string;          // e.g. "cortex.entropy.high"
        entropy: number;       // e.g. 0.95
        actionable: boolean;   // true if requires structural mutation
        content: string | object; 
    }
}
```

## 3. Autopoietic Mutation Examples (Cypher)

When `entropy > 0.85` is detected on the stream, `kernel/autopoiesis.js` executes:

### Crystallization (Node Creation)
```cypher
MERGE (r:BrainRegion {name: $sourceRegion})
MERGE (m:MemoryNode {id: $hash})
SET m.entropy = $entropy, m.content = $content, m.lastUpdated = timestamp()
MERGE (m)-[:SYNTHESIZED_BY]->(r)
```

### Pruning (Freeze State / Sleep Cycle)
```cypher
// Deletes nodes with high entropy that haven't crystallized after 24h
MATCH (m:MemoryNode)
WHERE m.entropy > 0.9 AND m.lastUpdated < timestamp() - 86400000
DETACH DELETE m
```
